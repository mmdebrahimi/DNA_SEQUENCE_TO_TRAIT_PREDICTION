Take a step back from the closed branch packets and rank the remaining ways to move the project forward.

- prospective validation channel exists

### Workhorse side

## Highest-value lanes

### 1. Source-control convergence recovery

Status:

- highest current value

Why:

- current biggest risk is silent divergence, not missing science packets
- decoder-side state is banked
- workhorse-side state still exists only as local report / packet files

Concrete value:

- makes cross-session state trustworthy
- prevents redoing closed work
- makes later branch selection cleaner

What counts as progress:

1. identify the intended tracked repo lineage for the workhorse artifacts
2. recover a real tracked clone or otherwise reattach this content to history
3. only then treat origin-backed state as genuinely converged

### 2. Decoder-side periodic prospective accrual

Status:

- valuable later, not urgent now

Why:

- the prospective channel is the cleanest way to keep adding evidence without changing the frozen decoder
- but near-term yield is low while the post-freeze window is still small

Concrete value:

- honest future-facing validation
- tamper-evident performance accrual after the freeze date

What counts as progress:

1. define a lightweight cadence
2. wait until enough post-freeze isolates exist
3. run the next periodic validation on a meaningful cohort, not a tiny trickle

### 3. External-label / independent-gold-set path for the decoder

Status:

- high strategic value
- externally gated

Why:

- the decoder platform is more mature than the current frontier science
- the cleanest next value may come from a stronger held-out label source, not another substrate support check

Concrete value:

- moves the project from internal closure to stronger external validation
- likely higher leverage than more no-money Path B work

What blocks it:

- proprietary access
- human curation
- money / contact / MTA constraints

### 4. Path B escalation contract

Status:

- worth doing only if there is appetite to continue Arabidopsis embedding work

Why:

- `AF-002` already did its support-check job
- default no-money reruns are now low-VOI
- the only honest Path B continuation is a declared escalation

Concrete value:

- prevents drift back into small mixed-signal runs
- forces success criteria before more compute is spent

What the contract must freeze:

1. compute tier
2. exact rerun packet
3. full metric surface
4. explicit stop condition

### 5. New-frontier intake

Status:

- likely the best pure science lane after convergence hygiene

Why:

- both current eukaryotic branches have largely paid out their no-money value
- fresh value probably requires a cleaner substrate, not deeper rework of mixed or closed branches

Concrete value:

- unlocks a new decision-bearing experiment
- avoids re-litigating already-decided packets

Only promote a new branch if all are true:

1. exact source is pinned
2. smallest credible experiment is checkable
3. time-to-first-packet is bounded
4. the branch changes a real portfolio decision now

### 6. Re-scoped fungal interpretation branch

Status:

- valid only under a new contract

Why:

- `NF-001` already falsified the old binary ERG11-only framing
- fungal work only becomes valuable again if the output contract changes

Concrete value:

- possible only if someone needs lineage-aware, non-binary interpretation

Required before reopening:

1. named downstream consumer
2. non-binary output contract
3. one precise beyond-`ERG11` question

## Recommended order

1. recover workhorse-side source-control convergence
2. keep default Path B closed unless an explicit escalation contract is written
3. schedule decoder prospective accrual later, not immediately
4. prepare for an externally gated gold-set / label path if access becomes available
5. otherwise select a genuinely new frontier with a bounded first packet

## What is not worth doing now

- more default FT16 support runs
- reopening `NF-001` under the old binary objective
- immediate prospective validation on a tiny post-freeze pool
- pretending the workhorse side is converged because local `.git` exists

## Short readout

The project is no longer bottlenecked on easy local experiments. The best remaining value is operational convergence first, then either a deliberate Path B escalation, a stronger external validation path, or a genuinely new frontier with a cleaner signal surface.
