## Problem Statement

v0 ships as a cached-strain cipro predictor (`pipeline.py predict --strain-id ...`). To advance the "AI DNA decoder tool" north star, v0.1 must EITHER:

- **Path G (genome-input):** accept a previously-unseen E. coli genome (FASTA, optionally GFF3) and emit the same v0 JSON+MD output. Closes the biggest gap between v0 + the original "decoder" concept.
- **Path C (cef-cached expansion):** ship a second cached-drug surface (ceftriaxone) reusing the v0 infrastructure. Smaller capability jump but reuses everything.

Both paths share a common ingestion-contract problem: **how does a new input map to the runtime surface that produces a v0-compliant prediction?** Path G needs novel ingestion code; Path C needs a cef-specific cohort + audit. This plan defines what "ingestion contract" means for each.

Success criterion: both paths have answered every contract question listed below before any code fires.

## Design decisions (covers BOTH paths)

### D1: Both paths share the v0 JSON output schema verbatim

**Decision:** v0.1 emits the same JSON keys + types as v0. Adding new keys is allowed; renaming/removing existing keys is NOT.

**Rationale:** Schema stability is the cheapest interop guarantee. v0 consumers (markdown sidecar renderer, downstream scripts, tests) should still work on v0.1 output.

**Trade-off:** Forces both paths to lean on the same provenance shape. Acceptable — provenance is general enough.

---

### D2: Both paths reuse the leakage-safe `leave_one_accession_out` CV strategy

**Decision:** Any new classifier trained for v0.1 (cef on Path C; or retrained cipro on Path G if needed) uses `leave_one_accession_out` CV by default. `build_cohort`'s accession-uniqueness assertion (committed 2026-05-22) already prevents the leakage class.

**Rationale:** Don't ship v0.1 with regressed CV integrity. The 2026-05-22 LESSON pins this as load-bearing.

**Trade-off:** None — accession-grouped CV is the right default.

---

### D3: Both paths defer top-K=10 mechanism-recovery as a HARD gate

**Decision:** v0.1 inherits v0's relocked spec (interpretability is exploratory, not a hard gate). Top-K=10 ≥ 50% known-mechanism recovery is a *target*, NOT a ship gate.

**Rationale:** Cipro failed this gate at v0; cef would likely fail too at v0.1 scale. Hard-gating it would block ship; soft-targeting it preserves honest reporting.

**Trade-off:** Risk of perpetually deferring interpretability rigor. Mitigation: report the recovery rate in provenance; flag explicitly when below target.

---

### D4: Path G's compute happens on Precision 7780 by default

**Decision:** Real-genome-input v0.1 CLI runs `python -m scripts.pipeline predict --genome-fasta X.fna --annotations Y.gff3 ...` BUT requires a GPU-equipped host. GTX 860M (this laptop) cannot fit NT v2 100M; Precision 7780 RTX 3500 Ada works.

**Rationale:** Hardware reality. NT v2 100M needs ≥ 6 GiB VRAM for inference; GTX 860M has 4 GiB and is Maxwell CC=5.0 (no bf16, no flash-attn).

**Trade-off:** Cross-machine workflow stays. Mitigation: make the contract explicit so users know where to run it. Add a CPU fallback path as v0.2 work (NT v2 100M CPU inference is ~10-100x slower but feasible for one genome).

---

### D5: Path C's cef cohort build is gated on the BV-BRC categorical-MIC feasibility check

**Decision:** Before building a cef N=150 cohort, run the BV-BRC strict-MIC 4-drug feasibility census (already exists at `scripts/bvbrc_strict_mic_4drug_census.py`) at the categorical-MIC bar (relaxed) specifically for cef. If cef yields < 50 R + 50 S at the categorical bar, halt Path C + reopen substrate choice.

**Rationale:** The 2026-05-18 census found NO drug clears N=150 per-class strict-MIC. Categorical-MIC bar may be different. Don't burn cef cohort-build work on a known-infeasible target.

**Trade-off:** Adds a feasibility check before code. ~30 min. Cheap.

---

### D6: Both paths preserve the audit-merge-JSON propagation discipline

**Decision:** `--audit-merge-json` stays required for canonical reporting in v0.1. `--allow-missing-audit` flag (per Codex's relocked v0 spec) propagates as non-canonical / debug-only.

**Rationale:** Honest-output discipline is the most distinctive feature of the decoder; don't sacrifice it for novel-input convenience.

**Trade-off:** Cef cohort needs its own merge-gate audit JSON. Add to Path C's prerequisites.

---

### D7: Path G's novel-genome scoring is per-genome, not per-cohort

**Decision:** Path G's CLI scores ONE genome per invocation. Batch scoring is v0.2+.

**Rationale:** Per-genome keeps the CLI simple + matches the v0 single-strain pattern. Batch is an obvious extension but adds complexity (parallel embedding, batch caching, partial-success semantics).

**Trade-off:** Throughput. For demos + one-off queries, per-genome is fine.

## Risk flags

- **R1 (HIGH): Path G's GTX 860M cannot run NT v2 100M.** Mitigation: D4 makes Precision 7780 the default GPU host; CPU fallback is v0.2 work. If Precision 7780 access is interrupted (corporate IT, hardware), Path G blocks.
- **R2 (MEDIUM): Path C's cef cohort may not clear the feasibility bar.** Mitigation: D5 runs the check first. If cef has < 50 R + 50 S at categorical-MIC, pivot to a 4th-drug candidate (mentioned in `wiki/ep1_ep2_cross_drug_architectural_finding_2026-05-17.md` falsification trigger).
- **R3 (MEDIUM): Both paths share the cross-machine-sync risk.** Mitigation: `scripts/cross_machine_sync_check.py` (shipped 2026-05-24) runs at each phase boundary; results emit drift signals.
- **R4 (LOW): Path G's GFF3 dependency is fragile.** If a user supplies FASTA-only, attribution can still run but with locus_tag-only — no gene-symbol matching to the mechanism catalog. Mitigation: emit a `partial_attribution=True` flag in JSON when GFF3 is absent.
- **R5 (LOW): v0.1G's "novel genome" claim is only as honest as the cohort the classifier was trained on.** If the input genome is from a clade not in N=147, attribution may be unreliable — this is where Mash-cluster (P2 reclassification) earns its keep.

## Open Questions

1. Should Path G's `--genome-fasta` accept multi-FASTA (multiple genomes per invocation)? Default: no — per-genome CLI per D7.
2. Should Path G persist novel-genome embeddings to the canonical cache? Default: no — transient only, to prevent accidental cache pollution. Promote via explicit `--persist` flag if user opts in (v0.2 work).
3. Should Path C's cef cohort overlap strain_ids with cipro N=147? Default: separate cohorts to avoid same-strain train/test pollution. If overlap is desired, the accession-uniqueness assertion catches it.
4. What's the v0.2 target after both paths land? Likely: 3rd drug + per-gene NT windows OR multi-organism portability. Defer to a future `/idea-anchor` cycle.
5. Should v0.1G's CLI surface a `--mash-cluster-context` flag that places the input genome in a pre-computed clade? Yes if Mash-cluster orchestration (P2 reclassification) lands first; v0.2 otherwise.
